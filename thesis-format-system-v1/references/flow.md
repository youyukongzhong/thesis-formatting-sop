# Thesis format flow

## Purpose

Run thesis-format work in a stable order:

`requirements -> diagnosis -> styles -> structure -> paragraphs -> figures/tables -> references/citations -> final check`

## 1. Read requirements first

Extract the hard rules before touching the thesis:

- page size and margins
- header and footer
- page-number rules
- abstract and keywords
- TOC depth
- heading hierarchy
- body font and spacing
- figure and table captions
- references
- in-text citation style

## 2. Diagnose the thesis

Inspect:

- file type
- sections
- page numbering
- headers/footers
- navigation
- TOC
- paragraph consistency
- figure/table condition
- references and citations
- blank pages, blank paragraphs, corrupt formatting

Record the issues before editing.

## 3. Build styles first

Normalize at least these styles:

- chapter heading
- heading 1/2/3
- body text
- abstract heading/body
- keywords
- TOC 1/2/3
- figure caption
- table caption
- references
- conclusion
- acknowledgements

## 4. Repair global structure

Before fine cleanup, fix:

- section breaks
- header/footer setup
- page-number restarts and continuity
- chapter starts on new pages
- heading hierarchy
- navigation
- automatic TOC

## 5. Normalize paragraphs

Then fix:

- font and size
- line spacing
- first-line indent
- spacing before/after
- alignment
- blank paragraphs
- broken wraps

## 6. Handle figures and tables

Check:

- figure number and title
- table number and title
- caption placement
- three-line table requirement
- screenshots that should be redrawn
- stray carriage returns or borders inside graphics

## 7. Handle references and citations

First normalize the references list, then the body citations:

- consistent reference entries
- numbering order
- superscript citations when required
- inline citation form when explicitly referenced in text
- no citations inside headings

## 8. Final check

Use `references/checklist.md` before delivery. Do not skip this step.
