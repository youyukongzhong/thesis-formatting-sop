# Guardrails

## Stay within scope

This system is for formatting, structure repair, and final verification first.

It may also touch adjacent problems only when the formatting task depends on them, such as:

- title wording cleanup
- figure/table caption completion
- reference formatting
- citation placement

## Do not silently cross these lines

Do not silently decide:

- to delete entire chapters
- to rewrite the core argument
- to invent data
- to fabricate references
- to claim a screenshot is an original figure
- to change a teacher's explicit direction

When such issues appear, flag them as `needs human confirmation`.

## Treat teacher feedback as high priority

When teacher comments conflict with the current thesis wording or with the default workflow:

1. preserve the evidence
2. follow the teacher's explicit formatting direction first
3. surface any remaining content-risk or data-risk items clearly

## Treat corrupted documents carefully

If the thesis is already corrupted:

- do not keep patching the dirtiest copy
- return to the cleanest safe backup
- rebuild styles and structure there

## Treat docx as the formal working format

If the incoming file is `.doc`, convert first and continue in `.docx`.
