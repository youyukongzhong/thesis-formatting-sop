# 论文修改系统 SOP v1

一套把论文格式修改从"凭经验硬改"变成"按流程执行"的方法论和工具集。

```
格式要求 → 文档体检 → 样式库 → 全局框架 → 标题与目录
→ 正文段落 → 图表 → 文献与引用 → 终检
```

这条顺序不能反。如果反了，目录修好又坏，页码改了又乱，前面手工改的格式会被后面的样式覆盖。

## 有什么

```

├── 02_终检清单.md               ← 11 大类 40+ 项，交付前逐条核对

└── workspace/                   ← 把论文和格式要求放这里
    └── _把论文和格式要求放在这里.md
```

## 怎么用

### 纯人工执行

读 `01_SOP.md`，按 9 步顺序操作，最后用 `02_终检清单.md` 逐项验收。

适合不想折腾脚本、只想按流程走一遍的人。

### 配合 AI 使用（Claude Code / Codex / 其他 agent）

把 repo 路径或文件内容给 AI，告诉它：

> 使用 thesis-format-system-v1 处理这篇论文，严格遵循 01_SOP.md 的执行顺序。

AI 会读取 SOP 流程、终检清单、服务边界，然后执行。

如果你用的是 Codex，`thesis-format-system-v1/` 下已经配好了 Skill 文件，可以直接引用 `$thesis-format-system-v1`。

### 用 PowerShell 半自动执行

```
.\format_thesis_sop_v1.ps1 -ThesisPath "workspace\你的论文.docx" -RequirementsPath "workspace\格式要求.doc"
```

**注意：** 脚本里的样式参数（字体、字号、行距）是对应某个学校的示例，你需要改成自己学校的实际要求。搜索脚本中的 `【待修改】` 标记位置。

前置条件：
- Windows 系统
- 已安装 Microsoft Word（脚本通过 COM 操作 Word）
- 论文主文件是 docx（doc 可以转，但不在 doc 上长期施工）

## 这套系统的设计原则

1. **先读规范，再动论文。** 不读格式要求就动手，改完一定会返工。
2. **先在副本上改，永远不动唯一原稿。**
3. **先建样式库，再批量套样式。** 逐段硬修是最慢的方式。
4. **先修大结构（分节、页眉、页码），再碰细节（字体、行距）。**
5. **先格式，后内容。** 格式问题自动改，内容判断问题只标记不替作者决定。
6. **不代写、不包过、不伪造数据、不伪造文献。**
